### فصل هجدهم: مسیریابی پیشرفته و رفع مشکلات رایج

---

#### بخش اول: مقدمه

پس از آشنایی با مفاهیم پایه‌ی مسیریابی در فصل هفتم و راه‌اندازی اولیه‌ی میکروتیک در فصل چهاردهم، حالا نوبت به مسیریابی پیشرفته و عیب‌یابی مشکلات رایج می‌رسد. در این فصل یاد می‌گیریم چگونه مسیرهای ثابت (Static Route) تعریف کنیم، مسیریابی پویا (Dynamic Routing) چیست، و چگونه مشکلات رایج شبکه را شناسایی و رفع کنیم.

---

#### بخش دوم: مسیریابی ثابت (Static Route)

**مسیریابی ثابت** یعنی شما به صورت دستی به روتر می‌گویید که بسته‌های مقصد یک شبکه‌ی خاص را به کدام روتر بعدی (Gateway) بفرستد.

**چه زمانی از Static Route استفاده می‌کنیم؟**
- زمانی که شبکه‌ی کوچکی داریم و تعداد مسیرها محدود است
- زمانی که می‌خواهیم مسیر دستی به یک شبکه‌ی خاص (مثلاً VPN) اضافه کنیم
- زمانی که نیاز به کنترل کامل روی مسیرها داریم

**مثال: اتصال دو شبکه‌ی محلی از طریق روتر**

فرض کنید دو ساختمان داریم که هر کدام یک شبکه‌ی جداگانه دارند:
- ساختمان الف: `192.168.1.0/24` با روتر `192.168.1.1`
- ساختمان ب: `192.168.2.0/24` با روتر `192.168.2.1`

این دو روتر از طریق یک لینک اختصاصی به هم متصل هستند. برای اینکه دستگاه‌های ساختمان الف بتوانند با ساختمان ب ارتباط برقرار کنند، باید یک مسیر ثابت در روتر الف اضافه کنیم.

**تنظیم در MikroTik (روتر الف):**
```
/ip route add dst-address=192.168.2.0/24 gateway=10.10.10.2
```
(آدرس `10.10.10.2` آدرس رابط روتر ب در لینک بین دو روتر است)

**مشاهده‌ی مسیرهای ثابت:**
- از منوی سمت چپ، **IP → Routes** را انتخاب کنید.
- مسیرهایی که **D** (Dynamic) ندارند، مسیرهای ثابت هستند.
- برای اضافه کردن مسیر جدید، روی **+** کلیک کنید.

---

#### بخش سوم: مسیریابی پویا (Dynamic Routing)

در شبکه‌های بزرگ، اضافه کردن دستی مسیرها غیرممکن است. مسیریابی پویا به روترها اجازه می‌دهد به طور خودکار مسیرها را یاد بگیرند و با تغییرات شبکه خود را تطبیق دهند.

**پروتکل‌های مسیریابی پویا:**

| پروتکل | کاربرد | ویژگی‌ها |
|--------|-------|----------|
| **OSPF** | شبکه‌های داخلی (سازمان‌ها) | مقیاس‌پذیر، سریع، محبوب |
| **RIP** | شبکه‌های کوچک | ساده اما کند |
| **BGP** | اینترنت و اتصال بین ارائه‌دهندگان | پروتکل اصلی اینترنت |

**تنظیم ساده OSPF در MikroTik:**
```
/routing ospf instance add name=default router-id=1.1.1.1
/routing ospf area add name=backbone area-id=0.0.0.0
/routing ospf interface add interface=ether1 area=backbone
/routing ospf network add network=192.168.1.0/24 area=backbone
```

#### بخش چهارم: رفع مشکلات رایج

**۱. مشکل: اینترنت قطع است (بدون اتصال به WAN)**

**علائم:**
- هیچ دستگاهی به اینترنت دسترسی ندارد
- چراغ WAN روی مودم خاموش یا قرمز است

**مراحل عیب‌یابی:**
1. بررسی کنید مودم روشن است و کابل‌ها سالم هستند.
2. در MikroTik، به **IP → DHCP Client** بروید و ببینید `ether1` (یا پورت WAN) آیا IP گرفته است.
3. اگر IP گرفته نشد، با ارائه‌دهنده تماس بگیرید.
4. اگر IP گرفته شده، از خط فرمان MikroTik دستور زیر را بزنید:
   ```
   /ping 8.8.8.8
   ```
   اگر پاسخ نداد، مشکل از اتصال WAN یا ارائه‌دهنده است.

**۲. مشکل: IP نگرفتن (مشکل DHCP)**

**علائم:**
- دستگاه‌ها به وای‌فای وصل می‌شوند اما IP نمی‌گیرند
- دستگاه‌ها IP با محدوده‌ی `169.254.x.x` می‌گیرند (APIPA)

**مراحل عیب‌یابی:**
1. در MikroTik، به **IP → DHCP Server** بروید.
2. مطمئن شوید DHCP Server فعال است و **Interface** درست انتخاب شده.
3. به تب **Leases** بروید. اگر دستگاه‌ها در لیست نیستند، یعنی درخواست به DHCP نمی‌رسد.
4. از خط فرمان MikroTik، دستور زیر را بزنید:
   ```
   /ip dhcp-server print
   /ip dhcp-server network print
   ```
5. مطمئن شوید محدوده‌ی آدرس‌ها (`address-pool`) پر نشده باشد.
6. اگر همه چیز درست است، دستگاه را ریستارت کنید.

**۳. مشکل: سرعت کم اینترنت**

**علائم:**
- اینترنت وصل است اما کند کار می‌کند
- برخی سایت‌ها سریع و برخی کند باز می‌شوند

**مراحل عیب‌یابی:**
1. از ابزار **Tools → Bandwidth Test** برای تست سرعت بین دو دستگاه MikroTik استفاده کنید.
2. از ابزار **Tools → Ping** با تعداد بسته‌ی زیاد تست کنید:
   ```
   /ping 8.8.8.8 count=100
   ```
   اگر درصد Loss بالا بود، مشکل از خط یا ارائه‌دهنده است.
3. اگر Loss صفر است اما سرعت کم است، احتمالاً محدودیت در Queue دارید:
   - به **Queue → Simple Queues** بروید و ببینید محدودیتی اعمال شده است.
4. از **Tools → Torch** برای مشاهده‌ی ترافیک لحظه‌ای استفاده کنید:
   - **Interface:** `ether1` یا پورت WAN
   - **Protocol:** `ip`
   - ببینید چه ترافیکی در حال عبور است

**۴. مشکل: برخی سایت‌ها باز نمی‌شوند**

**علائم:**
- برخی سایت‌ها باز می‌شوند، برخی نه
- پیام DNS error یا timeout می‌آید

**مراحل عیب‌یابی:**
1. ابتدا تست کنید مشکل از DNS است یا مسیریابی:
   ```
   /ping google.com
   /ping 8.8.8.8
   ```
   - اگر `8.8.8.8` پاسخ داد اما `google.com` نداد → مشکل DNS
   - اگر هیچکدام پاسخ نداد → مشکل اتصال
2. اگر مشکل DNS است، به **IP → DNS** بروید و DNSهای معتبر (مانند `8.8.8.8` و `1.1.1.1`) را تنظیم کنید.
3. اگر مشکل مسیریابی است، از **Tools → Traceroute** استفاده کنید:
   ```
   /tool traceroute google.com
   ```
   ببینید بسته در کدام گام متوقف می‌شود.

**۵. مشکل: دسترسی به صفحه‌ی مدیریت مودم اینترنت ماهواره‌ای**

**علائم:**
- مودم اینترنت ماهواره‌ای نصب شده اما نمی‌توان به صفحه‌ی مدیریت آن دسترسی پیدا کرد
- آدرس مدیریت مودم (مثلاً `192.168.100.1`) باز نمی‌شود

**مراحل عیب‌یابی:**
1. آدرس مدیریت مودم معمولاً با شبکه‌ی داخلی روتر تداخل دارد. اگر روتر شما از `192.168.1.0/24` استفاده می‌کند و مودم هم `192.168.100.1` است، مشکلی نیست.
2. اگر هر دو از یک محدوده استفاده می‌کنند (مثلاً هر دو `192.168.1.1`)، باید یکی را تغییر دهید.
3. برای دسترسی به مودم، یک مسیر ثابت اضافه کنید:
   ```
   /ip route add dst-address=192.168.100.0/24 gateway=ether1
   ```
4. اگر از NAT استفاده می‌کنید و مودم پشت روتر است، ممکن است نیاز به استثنا در فایروال داشته باشید.

#### بخش پنجم: ابزارهای عیب‌یابی در MikroTik

**۱. Ping**

ساده‌ترین ابزار برای تست اتصال.

```
/ping 8.8.8.8 count=10 size=1000
```

**۲. Traceroute**

برای مشاهده‌ی مسیر بسته‌ها تا مقصد.

```
/tool traceroute google.com
```

**۳. Torch**

برای مشاهده‌ی ترافیک لحظه‌ای یک پورت.

- از منوی سمت چپ، **Tools → Torch** را انتخاب کنید.
- **Interface:** پورت مورد نظر (مثلاً `ether1`)
- **Protocol:** `ip`
- روی **Start** کلیک کنید.

**۴. Bandwidth Test**

برای تست سرعت بین دو دستگاه MikroTik.

```
/tool bandwidth-test address=192.168.1.100 protocol=udp duration=10s
```

**۵. Packet Sniffer**

برای ضبط و تحلیل بسته‌ها (مشابه Wireshark).

```
/tool sniffer start
/tool sniffer stop
/tool sniffer save file-name=capture
```

**۶. Log (گزارش‌ها)**

برای مشاهده‌ی رویدادهای سیستم و خطاها.

- از منوی سمت چپ، **System → Log** را انتخاب کنید.
- برای فیلتر کردن، روی **Topics** کلیک کنید.

#### بخش ششم: کارگاه عملی

**هدف:** پیاده‌سازی مسیریابی ثابت و رفع مشکلات رایج.

**ابزار:** دو دستگاه MikroTik (یا شبیه‌ساز)، کامپیوتر.

**مراحل:**

**۱. تنظیم دو شبکه‌ی مجزا**

- روتر A را با IP `192.168.1.1/24` روی پورت `ether2` و IP `10.10.10.1/24` روی پورت `ether1` تنظیم کنید.
- روتر B را با IP `192.168.2.1/24` روی پورت `ether2` و IP `10.10.10.2/24` روی پورت `ether1` تنظیم کنید.
- دو روتر را با کابل به هم وصل کنید (ether1 به ether1).

**۲. اضافه کردن مسیر ثابت**

- روی روتر A، مسیر زیر را اضافه کنید:
  ```
  /ip route add dst-address=192.168.2.0/24 gateway=10.10.10.2
  ```
- روی روتر B، مسیر زیر را اضافه کنید:
  ```
  /ip route add dst-address=192.168.1.0/24 gateway=10.10.10.1
  ```

**۳. تست اتصال**

- از یک دستگاه در شبکه‌ی A، یک دستگاه در شبکه‌ی B را پینگ کنید:
  ```
  ping 192.168.2.10
  ```

**۴. شبیه‌سازی قطعی اینترنت**

- DHCP Client روی WAN را غیرفعال کنید.
- با `ping 8.8.8.8` تست کنید.
- با **IP → Routes** ببینید مسیر پیش‌فرض (0.0.0.0/0) وجود دارد یا نه.
- اگر مسیر پیش‌فرض وجود ندارد، دستی اضافه کنید:
  ```
  /ip route add dst-address=0.0.0.0/0 gateway=آدرس_مودم
  ```

**۵. شبیه‌سازی مشکل DHCP**

- DHCP Server را غیرفعال کنید.
- یک دستگاه را به شبکه وصل کنید. باید IP از محدوده‌ی `169.254.x.x` بگیرد.
- DHCP Server را دوباره فعال کنید.

**۶. عیب‌یابی با Log**

- به **System → Log** بروید.
- یک قانون فایروال اضافه کنید که ترافیک مشکوک را log کند:
  ```
  /ip firewall filter add chain=forward src-address=192.168.1.100 action=log log-prefix="SUSPICIOUS"
  ```

#### بخش هفتم: جمع‌بندی

| مشکل | ابزار تشخیص | راه‌حل |
|------|------------|--------|
| قطعی اینترنت | Ping, DHCP Client | بررسی WAN، تماس با ارائه‌دهنده |
| IP نگرفتن | DHCP Server, Leases | فعال کردن DHCP، بررسی محدوده |
| سرعت کم | Bandwidth Test, Torch | بررسی Queue، محدودیت ارائه‌دهنده |
| سایت‌ها باز نمی‌شوند | Ping, DNS, Traceroute | تنظیم DNS، بررسی مسیریابی |
| دسترسی به مودم | Route, Ping | اضافه کردن مسیر ثابت |
