### فصل چهاردهم: راه‌اندازی اولیه - از صفر تا یک پستخانه‌ی هوشمند

---

#### بخش اول: مقدمه

در فصل قبل با میکروتیک و RouterOS آشنا شدیم. در این فصل، قدم به قدم یاد می‌گیریم چگونه یک دستگاه میکروتیک را از حالت اولیه راه‌اندازی کنیم تا بتواند به عنوان یک پستخانه‌ی محلی هوشمند (روتر) کار کند.

---

#### بخش دوم: مفاهیم پایه قبل از شروع

قبل از شروع پیکربندی، باید با چند مفهوم کلیدی آشنا شویم:

**۱. WAN (Wide Area Network) - ورودی اینترنت**
- همان «دروازه‌ی شهر» در داستان نتلند
- پورتی که اینترنت از آن وارد می‌شود (اتصال به مودم یا ارائه‌دهنده)
- معمولاً از طریق DHCP یا تنظیمات ثابت (Static) از ارائه‌دهنده دریافت می‌شود

**۲. LAN (Local Area Network) - شبکه‌ی داخلی**
- همان «محله یا ساختمان» در داستان نتلند
- شبکه‌ای که دستگاه‌های داخل خانه یا اداره به آن متصل می‌شوند
- معمولاً از آدرس‌های خصوصی مثل `192.168.x.x` استفاده می‌کند

**۳. Bridge - پل ارتباطی**
- چند پورت را به هم متصل می‌کند تا مانند یک سوئیچ عمل کنند
- در روترهای خانگی، معمولاً همه‌ی پورت‌های LAN در یک Bridge قرار می‌گیرند

**۴. NAT (Network Address Translation) - ترجمه‌ی آدرس**
- آدرس خصوصی دستگاه‌های داخل شبکه را به آدرس عمومی روتر ترجمه می‌کند
- بدون NAT، دستگاه‌های داخل خانه نمی‌توانند به اینترنت متصل شوند

---

#### بخش سوم: مراحل راه‌اندازی اولیه

**مرحله ۱: اتصال فیزیکی و ورود به دستگاه**

1. دستگاه میکروتیک را به برق وصل کنید.
2. با کابل اترنت، پورت `ether1` را به کامپیوتر خود متصل کنید.
3. کامپیوتر خود را روی دریافت خودکار IP (DHCP) تنظیم کنید.
4. نرم‌افزار **WinBox** را اجرا کنید.
5. در تب **Neighbors**، دستگاه میکروتیک خود را پیدا کنید و روی آن دوبار کلیک کنید.
6. با نام کاربری `admin` و بدون رمز وارد شوید.

**مرحله ۲: ایجاد Bridge (پل ارتباطی)**

در روترهای خانگی، معمولاً همه‌ی پورت‌های LAN باید در یک Bridge قرار بگیرند تا مانند یک سوئیچ عمل کنند.

1. از منوی سمت چپ، **Bridge** را انتخاب کنید.
2. روی **+** (Add New) کلیک کنید.
3. یک نام دلخواه (مثلاً `bridge-local`) وارد کنید.
4. روی **OK** کلیک کنید.
5. حالا به تب **Ports** بروید.
6. روی **+** کلیک کنید و پورت‌های `ether2`، `ether3`، `ether4` (و هر پورت دیگری که برای LAN می‌خواهید) را به Bridge اضافه کنید.
7. برای هر پورت، **Interface** را انتخاب کنید و **Bridge** را همان `bridge-local` قرار دهید.

**مرحله ۳: تنظیم آدرس IP برای شبکه‌ی داخلی (LAN)**

حالا باید به Bridge خود یک آدرس IP بدهیم تا دستگاه‌های داخل خانه بتوانند به روتر متصل شوند.

1. از منوی سمت چپ، **IP** → **Addresses** را انتخاب کنید.
2. روی **+** (Add New) کلیک کنید.
3. **Address:** آدرس مورد نظر را وارد کنید (مثلاً `192.168.1.1/24`).  
   (علامت `/24` یعنی subnet mask برابر `255.255.255.0`)
4. **Interface:** `bridge-local` را انتخاب کنید.
5. روی **OK** کلیک کنید.

**مرحله ۴: راه‌اندازی DHCP Server (توزیع خودکار آدرس)**

برای اینکه دستگاه‌های داخل خانه بدون تنظیم دستی به شبکه متصل شوند، نیاز به DHCP Server داریم.

1. از منوی سمت چپ، **IP** → **DHCP Server** را انتخاب کنید.
2. روی **DHCP Setup** کلیک کنید.
3. **DHCP Server Interface:** `bridge-local` را انتخاب کنید.
4. **DHCP Address Space:** آدرس شبکه‌ی خود را وارد کنید (مثلاً `192.168.1.0/24`).
5. **Gateway for DHCP Network:** آدرس روتر را وارد کنید (مثلاً `192.168.1.1`).
6. **Addresses to Give Out:** محدوده‌ی آدرس‌هایی که به دستگاه‌ها داده می‌شود (مثلاً `192.168.1.10-192.168.1.254`).
7. **DNS Servers:** سرورهای DNS (مثلاً `8.8.8.8` و `1.1.1.1`).
8. روی **Next** کلیک کنید تا پایان. روی **OK** کلیک کنید.

**مرحله ۵: تنظیم WAN (ورودی اینترنت)**

حالا باید پورتی که اینترنت از آن وارد می‌شود را تنظیم کنیم. معمولاً `ether1` به مودم یا ارائه‌دهنده متصل می‌شود.

1. از منوی سمت چپ، **IP** → **DHCP Client** را انتخاب کنید.
2. روی **+** (Add New) کلیک کنید.
3. **Interface:** `ether1` را انتخاب کنید.
4. **Use Peer DNS:** تیک بزنید (تا DNS از ارائه‌دهنده گرفته شود).
5. **Add Default Route:** تیک بزنید (تا مسیر پیش‌فرض به طور خودکار اضافه شود).
6. روی **OK** کلیک کنید.

اگر ارائه‌دهنده‌ی شما آدرس ثابت (Static IP) می‌دهد، به جای DHCP Client، از **IP → Addresses** استفاده کنید و آدرس ثابت را وارد کنید.

**مرحله ۶: تنظیم NAT (ترجمه‌ی آدرس) برای اشتراک اینترنت**

برای اینکه دستگاه‌های داخل شبکه بتوانند به اینترنت متصل شوند، نیاز به NAT داریم.

1. از منوی سمت چپ، **IP** → **Firewall** را انتخاب کنید.
2. به تب **NAT** بروید.
3. روی **+** (Add New) کلیک کنید.
4. در تب **General**:
   - **Chain:** `srcnat` را انتخاب کنید.
   - **Src. Address:** آدرس شبکه‌ی داخلی خود را وارد کنید (مثلاً `192.168.1.0/24`).
5. در تب **Action**:
   - **Action:** `masquerade` را انتخاب کنید.
6. روی **OK** کلیک کنید.

**مرحله ۷: تنظیم DNS (اختیاری)**

اگر از DNSهای عمومی می‌خواهید استفاده کنید:

1. از منوی سمت چپ، **IP** → **DNS** را انتخاب کنید.
2. **Servers:** آدرس‌های DNS مورد نظر را وارد کنید (مثلاً `8.8.8.8,1.1.1.1`).
3. **Allow Remote Requests:** تیک بزنید تا دستگاه‌های داخل شبکه بتوانند از DNS روتر استفاده کنند.
4. روی **OK** کلیک کنید.

**مرحله ۸: تنظیم رمز عبور برای امنیت**

برای جلوگیری از دسترسی غیرمجاز، حتماً رمز عبور برای کاربر admin تنظیم کنید.

1. از منوی سمت چپ، **System** → **Users** را انتخاب کنید.
2. روی کاربر `admin` دوبار کلیک کنید.
3. روی **Password** کلیک کنید.
4. رمز عبور جدید را وارد کنید و تأیید کنید.
5. روی **OK** کلیک کنید.

#### بخش چهارم: جمع‌بندی تنظیمات

پس از انجام مراحل بالا، روتر شما آماده‌ی استفاده است. خلاصه‌ی تنظیمات:

| بخش | تنظیم |
|------|-------|
| **Bridge** | `bridge-local` شامل پورت‌های ether2 تا ether4 |
| **LAN IP** | `192.168.1.1/24` روی bridge-local |
| **DHCP** | محدوده‌ی `192.168.1.10-192.168.1.254` |
| **WAN** | DHCP Client روی ether1 |
| **NAT** | Masquerade برای شبکه‌ی `192.168.1.0/24` |
| **DNS** | `8.8.8.8` و `1.1.1.1` |

#### بخش پنجم: کارگاه عملی

**هدف:** راه‌اندازی یک میکروتیک از صفر تا آماده‌سازی برای استفاده.

**ابزار:** یک دستگاه MikroTik (مانند hAP یا RB750)، کامپیوتر با WinBox.

**مراحل عملی:**

**۱. ریست کردن دستگاه به تنظیمات کارخانه (اگر قبلاً استفاده شده)**

- در WinBox به دستگاه متصل شوید.
- از منوی سمت چپ، **System** → **Reset Configuration** را انتخاب کنید.
- گزینه‌ی **No Default Configuration** را انتخاب کنید (تا تنظیمات پیش‌فرض حذف شود).
- تیک **Do Not Backup** را بزنید.
- روی **Reset Configuration** کلیک کنید.
- دستگاه ریست می‌شود و دوباره با admin بدون رمز در دسترس خواهد بود.

**۲. انجام مراحل گفته شده**

- مراحل ۱ تا ۸ را به ترتیب انجام دهید.

**۳. تست اتصال**

- یک کامپیوتر یا گوشی را به یکی از پورت‌های LAN (مثلاً ether2) متصل کنید.
- بررسی کنید که IP خودکار گرفته باشد (مثلاً `192.168.1.x`).
- در خط فرمان کامپیوتر، دستور `ping 192.168.1.1` را بزنید. باید پاسخ دریافت کنید.
- دستور `ping 8.8.8.8` را بزنید. اگر پاسخ دریافت کردید، یعنی اینترنت وصل است.
- دستور `ping google.com` را بزنید. اگر پاسخ دریافت کردید، یعنی DNS درست کار می‌کند.

**۴. ذخیره‌ی تنظیمات (Backup)**

- از منوی سمت چپ، **Files** را باز کنید.
- روی **Backup** کلیک کنید.
- یک نام برای فایل پشتیبان وارد کنید و روی **Backup** کلیک کنید.
- فایل `.backup` در لیست فایل‌ها ظاهر می‌شود. می‌توانید آن را با **Download** روی کامپیوتر ذخیره کنید.

**۵. مشاهده‌ی آمار و اطلاعات**

- **Interfaces:** ببینید کدام پورت‌ها فعال هستند و ترافیک عبوری چقدر است.
- **IP → Addresses:** آدرس‌های تنظیم شده را ببینید.
- **IP → DHCP Server → Leases:** ببینید چه دستگاه‌هایی به شبکه متصل شده‌اند.
- **System → Resources:** میزان استفاده از CPU و RAM را ببینید.

#### بخش ششم: نکات مهم

| نکته | توضیح |
|------|-------|
| **پورت WAN** | معمولاً `ether1` به مودم یا فیبر نوری متصل می‌شود |
| **پورت‌های LAN** | بقیه پورت‌ها به کامپیوترها، سوئیچ‌ها یا اکسس پوینت‌ها متصل می‌شوند |
| **IP ثابت vs DHCP** | اگر ارائه‌دهنده IP ثابت می‌دهد، به جای DHCP Client از Addresses استفاده کنید |
| **پشتیبان‌گیری** | همیشه پس از تنظیمات، یک Backup بگیرید تا در صورت مشکل سریع برگردید |
| **به‌روزرسانی** | قبل از راه‌اندازی، از **System → Packages** مطمئن شوید RouterOS به‌روز است |
