برای درک بهتر نحوه عملکرد این سیستم، می‌توان آن را به دو بخش اصلی تقسیم کرد: آنچه در کامپیوتر شما (Local) اتفاق می‌افتد و آنچه در سرورهای ابری (Cloud) پردازش می‌شود.

### ۱. چه چیزی کجا دانلود و اجرا می‌شود؟

*   **روی سیستم شما (Local):**
    *   **دانلود:** شما فقط ابزار **Claude Code** را (که یک رابط خط فرمان یا CLI است) از طریق دستوراتی مثل `npm` یا `curl` دانلود و نصب می‌کنید [۱۳، ۲۲].
    *   **اجرا:** این ابزار مستقیماً در ترمینال یا سیستم‌عامل شما (ویندوز، مک یا لینوکس) اجرا می‌شود [۳، ۲۳]. این بخش وظیفه دارد فایل‌های شما را بخواند، کدها را ویرایش کند و دستورات ترمینال را در محیط سیستم شما اجرا کند [۱۱].
*   **روی سرورهای پردازش دیتا (Cloud):**
    *   **دانلود:** در این روش، **هیچ مدل هوش مصنوعی سنگینی روی سیستم شما دانلود نمی‌شود** [۱، ۲۰]. 
    *   **اجرا:** تمام فرآیند "فکر کردن" و تولید پاسخ توسط مدل‌های هوش مصنوعی (مانند Qwen یا GPT) روی سرورهای **OpenRouter** و ارائه‌دهندگان آن مدل‌ها انجام می‌شود [۲، ۲۱].

### ۲. تقسیم وظایف بین سیستم و سرور

| بخش | محل انجام (کجا؟) | وظیفه اصلی |
| :--- | :--- | :--- |
| **رابط کاربری و مدیریت فایل** | سیستم شما (Local) | خواندن کدهای پروژه، اعمال تغییرات در فایل‌ها، و اجرای دستورات در ترمینال شما [۱۱، ۲۶]. |
| **تنظیمات (Settings.json)** | سیستم شما (Local) | فایلی که در آن مشخص می‌کنید هوش مصنوعی باید از کجا (OpenRouter) دستور بگیرد [۴، ۲۷]. |
| **پردازش و تولید کد (LLM)** | سرورهای ابری (Cloud) | تحلیل سوال شما و فرستادن دستورات لازم برای حل مشکل یا نوشتن کد جدید [۲، ۲۱]. |

### ۳. توضیح مفاهیم به زبان ساده

*   **OpenRouter (اوپن‌روتر):** مانند یک "مرکز تلفن" یا واسطه عمل می‌کند. به جای اینکه مجبور باشید برای هر هوش مصنوعی جداگانه پول بدهید، با یک کلید به صدها مدل مختلف (رایگان و پولی) متصل می‌شوید [۲، ۲۱].
*   **CLI (رابط خط فرمان):** ابزاری است که به جای منوهای گرافیکی، با تایپ کردن دستور در ترمینال با آن کار می‌کنید [۱۳، ۲۲].
*   **API Key (کلید ای‌پی‌آی):** یک رمز عبور مخصوص است که به OpenRouter می‌گوید شما اجازه دارید از خدمات آن‌ها استفاده کنید [۳، ۲۳].
*   **Agentic (عامل‌بودن):** به این معنی است که هوش مصنوعی فقط حرف نمی‌زند، بلکه می‌تواند مثل یک برنامه نویس واقعی، فایل‌ها را باز کند، کد بزند و برنامه‌ها را تست کند [۶، ۱۱].
*   **Settings.json:** یک فایل متنی ساده است که تنظیمات برنامه در آن ذخیره می‌شود؛ مثلاً در اینجا آدرس OpenRouter را در آن وارد می‌کنید تا Claude Code بداند از کدام مدل استفاده کند [۴، ۸].

با این روش، شما قدرت یک ابرکامپیوتر را برای کدنویسی در اختیار دارید، در حالی که سیستم خودتان اصلاً تحت فشار قرار نمی‌گیرد [۱، ۲۶]. اگر تمایل دارید، می‌توانم در مورد تنظیمات دقیق فایل **settings.json** برای اتصال به یک مدل رایگان خاص بیشتر توضیح دهم.


برای راه‌اندازی این ابزار به صورت **پروژه محور** (Local)، به گونه‌ای که فقط برای یک پروژه خاص از مدل‌های رایگان یا ارزان استفاده کند، مراحل زیر را دنبال کنید:

### ۱. نصب Claude Code
ابتدا باید خودِ ابزار Claude Code را روی سیستم نصب کنید. بسته به سیستم‌عامل خود از دستورات زیر در ترمینال استفاده کنید:

*   **نصب عمومی (توصیه شده):**
    `npm install -g @anthropic-ai/claude-code` [۱۳، ۲۲]
*   **برای مک و لینوکس (جایگزین):**
    `curl -fsSL https://claude.ai/install.sh | sh` [۱۳، ۲۲]
*   **برای ویندوز (PowerShell):**
    `irm https://claude.ai/install.ps1 | iex` [۱۳]

### ۲. دریافت کلید API از OpenRouter
1.  به سایت **OpenRouter** بروید و یک حساب کاربری بسازید [۴، ۲۳].
2.  در بخش **API Keys**، یک کلید جدید ایجاد کرده و آن را کپی کنید (این کلید معمولاً با `sk-or-` شروع می‌شود) [۱۴، ۲۳].
3.  نام یک مدل رایگان (مانند `google/gemini-2.0-flash-001` یا `openrouter/free-test-model`) را از لیست مدل‌های OpenRouter انتخاب و کپی کنید [۲، ۶، ۲۴].

### ۳. تنظیمات پروژه (پیکربندی محلی)
برای اینکه تنظیمات فقط روی پروژه فعلی شما اعمال شود، این مراحل را در پوشه اصلی پروژه انجام دهید:
1.  یک پوشه به نام **`.claude`** بسازید [۴، ۲۳].
2.  داخل این پوشه، فایلی به نام **`settings.json`** ایجاد کنید [۴، ۲۳].

### ۴. متن فایل کانفیگ (`settings.json`)
محتویات زیر را عیناً در فایل `settings.json` کپی کرده و در قسمت‌های مشخص شده، کلید API و نام مدل خود را قرار دهید:

```json
{
  "ANTHROPIC_BASE_URL": "https://openrouter.ai/api/v1",
  "ANTHROPIC_API_KEY": "",
  "ANTHROPIC_O_TOKEN": "YOUR_OPENROUTER_API_KEY",
  "ANTHROPIC_MODEL": "YOUR_SELECTED_MODEL_NAME"
}
```
**نکات بسیار مهم:**
*   مقدار **`ANTHROPIC_API_KEY`** حتماً باید **خالی** (`""`) بماند تا ابزار به جای سرورهای اصلی Anthropic، به OpenRouter متصل شود [۴، ۱۴، ۲۴].
*   در قسمت **`ANTHROPIC_O_TOKEN`**، کلید API که از OpenRouter گرفتید را وارد کنید [۴، ۱۴].
*   در قسمت **`ANTHROPIC_MODEL`**، نام دقیق مدلی که انتخاب کردید را بنویسید [۵، ۲۴].

### ۵. اجرا و تست
پس از ذخیره فایل، در ترمینال و در همان مسیر پروژه، دستور زیر را تایپ کنید:
`claude` [۷، ۲۵]

اگر تنظیمات درست باشد، Claude Code با استفاده از مدل انتخابی شما در OpenRouter بالا می‌آید [۸، ۲۵]. برای اطمینان از اینکه از مدل درستی استفاده می‌شود، می‌توانید از دستور **/models** در محیط برنامه استفاده کنید تا مدل فعال را مشاهده کنید [۱۶، ۲۶].

**پیشنهاد:** اگر با خطای ۴۰۲ (کمبود اعتبار) مواجه شدید، مطمئن شوید که یک مدل کاملاً رایگان را انتخاب کرده‌اید یا حساب OpenRouter خود را مقدار کمی شارژ کرده‌اید [۵، ۱۴]. برای عملکرد بهتر در کدنویسی، مدل‌های **MiniMax 2.5** یا **Step 3.5 Flash** پیشنهاد می‌شوند [۶، ۹، ۱۰].
