## اصل ماجرا: Qwen کجاست؟

اول بدونیم **Qwen کجا اجرا می‌شود**. دو حالت دارد:

| حالت | Qwen کجاست؟ | نیاز به API Key |
| :--- | :--- | :--- |
| **اجرای آنلاین (API)** | روی سرورهای علی‌بابا (یا سرویس‌های دیگر) | ✅ نیاز دارد |
| **اجرای آفلاین (محلی)** | روی کامپیوتر خودتان | ❌ نیاز ندارد |

---

## چرا Qwen به API Key نیاز دارد؟

**دلیل اصلی: پرداخت هزینه اجرای مدل روی سرورها**

وقتی شما از Qwen سوال می‌پرسید، این اتفاق می‌افتد:

```
شما (کامپیوتر) --> اینترنت --> سرورهای قوی (GPU های گران قیمت) --> پردازش سوال --> جواب برمی‌گردد
```

نگهداری این سرورها برای شرکت **هزینه دارد** (برق، سخت‌افزار، نگهداری، پهنای باند). API Key مثل یک **کارت شناسایی** است که به شرکت می‌گوید: «این شخص دارد از خدمات ما استفاده می‌کند».

---

## API Key چه فایده‌ای برای Qwen دارد؟

از دیدگاه Qwen (شرکت)، API Key سه فایده اصلی دارد:

### 1. کنترل مصرف (مثل قبض آب و برق)
بدون API Key، هرکسی می‌توانست بی‌نهایت سوال بپرسد و هزینه‌های شرکت را بالا ببرد. با API Key:
- می‌توانند ببینند هر کاربر چقدر استفاده می‌کند
- اگر کسی بیش از حد استفاده کند، می‌توانند محدودش کنند یا ازش پول بگیرند

### 2. جلوگیری از سوءاستفاده
اگر API Key نبود، هکرها می‌توانستند میلیون‌ها درخواست همزمان بفرستند و سرورها را از کار بیندازند. API Key مثل **قفل درب خانه** است.

### 3. مدل کسب‌وکار (پول درآوردن)
شرکت‌ها می‌توانند:
- **نسخه رایگان** بدهند با محدودیت (مثلاً روزی 100 سوال)
- **نسخه پولی** بدهند بدون محدودیت
- به توسعه‌دهندگان اجازه بدهند از Qwen در نرم‌افزارهای خود استفاده کنند

---

## پس چرا بعضی از مدل‌ها (مثل Ollama) به API Key نیاز ندارند؟

**چون روی کامپیوتر خودتان اجرا می‌شوند!**

مقایسه کنید:

| | Qwen Code (با API Key) | Ollama (بدون API Key) |
| :--- | :--- | :--- |
| **مدل کجا اجرا می‌شود؟** | روی سرورهای علی‌بابا | روی کامپیوتر خودتان |
| **نیاز به اینترنت؟** | بله (همیشه) | فقط برای دانلود اولیه |
| **سرعت** | خیلی سریع (سرورهای قوی) | بستگی به کامپیوتر شما دارد |
| **محدودیت** | دارد (مگر پول بدهید) | هیچ محدودیتی |
| **نیاز به سخت‌افزار قوی؟** | نه | بله (GPU خوب می‌خواهد) |

---

## مثال ساده برای درک بهتر

فرض کنید می‌خواهید یک **ماشین حساب خیلی قوی** داشته باشید:

- **روش API Key (Qwen Code)**: ماشین حساب در مغازه است. شما هر بار که می‌خواهید چیزی حساب کنید، به مغازه می‌روید. صاحب مغازه از شما می‌پرسد «کدت رو بده ببینم چند بار استفاده کردی». (اگر زیاد بیایی، ممکنه ازت پول بگیره)

- **روش آفلاین (Ollama)**: ماشین حساب را می‌خرید می‌برید خانه. هرچقدر دوست دارید استفاده می‌کنید، کسی هم از شما نمی‌پرسد کدت چیست. ولی باید هزینه خرید ماشین حساب را بپردازید (یعنی کامپیوتر قوی داشته باشید).

## جمع‌بندی نهایی

**API Key یعنی:**
- شما از سرورهای شرکت استفاده می‌کنید (نیازی به کامپیوتر قوی ندارید)
- شرکت می‌تواند مصرف شما را کنترل کند
- برای استفاده رایگان معمولاً محدودیت دارد

**دلیل استفاده Qwen از API Key:**
1. 💰 **کسب درآمد** (از کاربران پولی)
2. 🛡️ **امنیت** (جلوگیری از حمله و سوءاستفاده)
3. 📊 **مدیریت منابع** (مشخص کردن سهم هر کاربر)

## توصیه به شما

اگر **کامپیوتر قوی** داری (مثلاً کارت گرافیک 8 گیگ یا بیشتر) و **فضای خالی** خوبی داری → برو سراغ **Ollama** (رایگان، بدون API Key، آفلاین)

اگر **کامپیوتر معمولی** داری یا می‌خواهی همیشه جدیدترین نسخه را داشته باشی → از **API Key با OpenRouter** استفاده کن (رایگان با محدودیت روزانه)
