# چک‌لیست مرور مستندات

**یک خط:** قبل از اینکه بگویی «آماده‌ست» یا به AI بگویی «پیاده کن»، این لیست را برو — مثل وقتی قبل از تحویل کار به مشتری یک بار دیگر با چشم تازه می‌خوانی.

چک‌لیست جای نوشتن داک نیست؛ جای **کنترل کیفیت** است. اگر جواب بیشتر «نه» است، برگرد به راهنمای همان سند — [how-to-write-proposal.md](./how-to-write-proposal.md) و بقیه.

---

## لحن و سبک — همهٔ `docs/` (فارسی)

این بخش برای **همه** سندهای فارسی در `docs/` است — proposal، design، ui-behavior، tasks.

- [ ] با **داستان یا مثال واقعی** شروع شده — نه جدول خالی و نه «§۱ مقدمه»؟
- [ ] کمتر از **سه جدول** در کل سند (مگر design واقعاً لیست API کوتاه لازم دارد)؟
- [ ] جمله‌ها **کامل**‌اند — نه تلگراف مثل «ریسک XSS / CSP / iframe»؟
- [ ] یک دوست غیرفنی بعد از خواندن **بدون سوال فنی** می‌فهمد موضوع چیست؟
- [ ] متن شبیه **spec انگلیسی ترجمه‌شده** نیست؟
- [ ] فایل فارسی: `<div dir="rtl">` در **ابتدا** و بلوک `<style>` کامل در **انتهای** فایل؟

مرجع لحن: [how-to-write-proposal.md](./how-to-write-proposal.md) بخش «وقتی AI می‌نویسد».

---

## proposal.md

- [ ] **عنوان** و **یک خط** زیر آن وجود دارد؟
- [ ] با **مثال ملموس** شروع شده — «فرض کن ادمین روی صفحهٔ اصلی است…»؟
- [ ] **مشکل** و **مخاطب** روشن‌اند — با پاراگراف، نه فقط bullet؟
- [ ] گفته شده **چه می‌کند** و **چه نمی‌کند** (خارج از scope)؟
- [ ] **معیار موفقیت** قابل فهم و قابل چک است — «ادمین بدون SSH بتواند…»؟
- [ ] **جزئیات فنی** (stack، API، نام فایل) قاطی نشده؟
- [ ] فاز ۱ و فاز بعد **جدا** توضیح داده شده‌اند؟
- [ ] برای بخش‌های مهم **ریسک سطح محصول** گفته شده — با جملهٔ کامل؟
- [ ] حداقل **یک مسیر E2E حیاتی** نام برده شده (مثلاً `E2E-P-01`)؟
- [ ] یک دوست غیرفنی می‌فهمد «این برای چیه»؟

---

## design.md

- [ ] معلوم است این سند درباره **کدام بخش** سیستم است؟
- [ ] **محدوده** — داخل و خارج — روشن است؟
- [ ] **تصویر کلی** در یک یا دو پاراگراف (یا دیاگرام ساده) هست؟
- [ ] اجزا با «می‌کند / نمی‌کند» تعریف شده‌اند — نه فقط لیست اسم؟
- [ ] حداقل **یک فرایند** با مثال واقعی و خطا نوشته شده؟
- [ ] برای فرایندهای مهم **«اگر شکست خورد»** آمده؟
- [ ] **ورودی‌های حساس** و **لاگ** (چه چیزی لاگ نشود) یادآوری شده؟
- [ ] **وضعیت‌ها** و **خطاها** از دید سیستم مشخص‌اند؟
- [ ] **تصمیم‌های عمدی** («چرا این راه را انتخاب کردیم») ثبت شده؟
- [ ] چیزی از proposal **تکرار** نشده — فقط ارجاع؟
- [ ] **ریسک فنی** + راه کاهش آمده؟
- [ ] **سناریوهای E2E** با پیش‌شرط و شماره (مثلاً `E2E-D-01`)؟
- [ ] جزئیات **دکمه و toast** اینجا نیست — آن‌ها ui-behavior است؟

اگر design از حدود ۴۰۰ خط گذشت → بخش‌های موضوعی به فایل جدا منتقل شده؟

---

## ui-behavior.md

- [ ] با **داستان** نوشته شده — نه فقط لیست ویژگی؟
- [ ] **هر صفحهٔ مهم:** ورود، اولین نگاه، کارها، loading، empty، خطا، موفق؟
- [ ] **جریان‌های چندمرحله‌ای** شاخه‌به‌شاخه (انصراف، خطا، موفق)؟
- [ ] **نمونهٔ جملهٔ خطا** برای خطاهای رایج — انسانی، نه کد HTTP؟
- [ ] **دسترسی‌پذیری** — Tab، Enter — در حد لازم گفته شده؟
- [ ] **موبایل** — چه تفاوتی حس می‌شود (بدون پیکسل)؟
- [ ] **خروج بدون ذخیره** در فرم‌های طولانی؟
- [ ] چیزی از design **تکرار** نشده — فقط رفتار روی صفحه؟
- [ ] جریان‌های حیاتی برای **E2E** علامت‌گذاری شده‌اند؟

اگر از حدود ۴۰۰ خط گذشت → فایل‌های فرعی مثل `ui-behavior-move-card-view.md`؟

---

## tasks.md

- [ ] با **زبان ساده** نوشته شده؟
- [ ] هر کار **قابل تیک** و اندازهٔ معقول است؟
- [ ] **ارجاع به design** آمده — «طبق بخش جابه‌جایی با کشیدن»؟
- [ ] فازها **به ترتیب منطقی** — از design آمده‌اند؟
- [ ] **ریسک** در ابتدای فازهای حساس یادآوری شده؟
- [ ] **وابستگی** کارها مشخص است — «بعد از فاز ۲»؟
- [ ] **تست** (دستی / خودکار / E2E) کنار کارهای مهم؟
- [ ] **تموم شدن فاز** برای هر فاز معنی دارد؟
- [ ] design/proposal **تکرار** نشده؟

## change/ و archive/

- [ ] پوشهٔ باز **بدون تاریخ** — `docs/change/cart-view/`؟
- [ ] پوشهٔ تموم‌شده **با تاریخ شمسی** — `1405-03-01-cart-view/`؟
- [ ] داخل change: proposal + design + tasks (و ui-behavior اگر UI دارد)؟
- [ ] archive فقط برای **رد شده / معلق** — نه کار تموم‌شده؟

## یک سوال سریع برای هر سند

**proposal** — «یک دوست غیرفنی می‌فهمد این برای چیه؟»

**design** — «یک توسعه‌دهندهٔ جدید می‌فهمد چه اتفاقی می‌افتد — بدون کد؟»

**ui-behavior** — «یک تستر می‌فهمد روی صفحه چه باید ببیند؟»

**tasks** — «می‌دانی قدم بعدی چیست و کی فاز تمام می‌شود؟»

اگر به هر چهار سوال «بله» گفتی، نزدیک «آماده‌ست» هستی.
