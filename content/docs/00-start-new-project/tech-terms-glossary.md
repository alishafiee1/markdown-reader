# اصطلاح‌نامه فنی — معادل درست فارسی

**یک خط:** وقتی در `docs/` می‌نویسی یا AI ترجمه می‌کند، این فایل می‌گوید **چه کلمه‌ای درست است** — تا «تصویر Docker» به‌جای image نیاید و خواننده فکر نکند منظور عکس است.

این فایل استثناست: **جدول اینجا ابزار کار است** — نه سبک تلگرافی. ولی مقدمه و توضیح استفاده را با همان لحن خودمونی بقیهٔ پوشه نگه می‌داریم.

**قانون کلی:** اگر معادل فارسی رایج و قابل فهم ندارد، اصطلاح انگلیسی را نگه دار و در پرانتز یک جملهٔ ساده بزن.

**در پروپزال و ui-behavior:** تا جایی می‌توانی اصلاً اصطلاح نیاوری — «بستهٔ نصب Docker» به‌جای «ایمیج». در design و docs-for-ai اصطلاح دقیق‌تر مجاز است.

---

## بخش ۱ — Docker و زیرساخت

| اصطلاح | ترجمهٔ نادرست رایج | معادل پیشنهادی | توضیح ساده |
|--------|-------------------|----------------|------------|
| `image` (Docker) | تصویر | **ایمیج Docker** | بسته‌ای آماده که از Dockerfile ساخته می‌شه — مثل قالب نصب |
| `container` | ظرف / مخزن | **کانتینر** | یک برنامهٔ در حال اجرا که از روی ایمیج ساخته شده |
| `registry` | دفتر ثبت | **رجیستری ایمیج** | جایی مثل Docker Hub که ایمیج‌ها ذخیره می‌شن |
| `build` (image) | ساخت | **build کردن** | دستوری که از Dockerfile یک ایمیج می‌سازه |
| `deploy` | مستقر کردن / استقرار | **deploy کردن** (یا انتشار روی سرور) | فرستادن برنامه به سرور و روشن کردنش — یک انتخاب کن، همیشه همون رو بنویس |
| `flag` (boolean) | پرچم | **فلگ** یا **گزینه** | یک تنظیم روشن/خاموش — مثل «حالت debug» |
| `serve` (HTTP) | خواندن | **ارائه دادن** یا **سرو کردن** | وقتی سرور یه فایل یا صفحه رو به مرورگر می‌فرسته |
| `egress` | خروجی | **ترافیک خروجی** | داده‌ای که از سرور به بیرون می‌ره |
| `ingress` | ورودی | **ترافیک ورودی** | درخواست‌هایی که از بیرون به سرور می‌رسن |
| `orchestration` | هماهنگی | **هماهنگ‌سازی کانتینر** | مدیریت خودکار چند کانتینر (مثل Kubernetes) |
| `host` (server) | میزبان | **سرور میزبان** | کامپیوتری که برنامه روش اجرا می‌شه |
| `runtime` | زمان اجرا | **محیط اجرا** | چیزی که برنامه برای کار کردن بهش نیاز داره — مثل Node.js |

## بخش ۲ — Git و توسعه

| اصطلاح | ترجمهٔ نادرست رایج | معادل پیشنهادی | توضیح ساده |
|--------|-------------------|----------------|------------|
| `commit` | ثبت / ارسال | **کامیت** | ذخیرهٔ یک تغییر در تاریخچهٔ git |
| `branch` | شاخه | **شاخه** (قابل قبول) یا **branch** | نسخهٔ موازی از کد برای کار جدا |
| `pull request` | درخواست تغییر | **PR** یا **Pull Request** | درخواست ادغام کد جدید به شاخهٔ اصلی |
| `merge` | ادغام | **ادغام** (قابل قبول) | وصل کردن دو شاخه به هم |
| `release` | انتشار | **release** یا **انتشار نسخه** | نسخهٔ رسمی که به کاربر می‌ره |
| `breaking change` | تغییر شکننده | **تغییر ناسازگار** | تغییری که کد قبلی رو خراب می‌کنه |
| `rollback` | بازگشت | **برگشت به نسخهٔ قبل** | رفتن به نسخهٔ قبلی وقتی چیزی خراب شد |
| `revert` | معکوس کردن | **لغو تغییر** | برگشت یک کامیت مشخص |
| `tag` (Git) | برچسب | **تگ نسخه** | علامتی روی یک کامیت که نسخه رو مشخص می‌کنه (مثل v1.2.0) |

## بخش ۳ — API و ارتباط

| اصطلاح | ترجمهٔ نادرست رایج | معادل پیشنهادی | توضیح ساده |
|--------|-------------------|----------------|------------|
| `endpoint` | نقطهٔ انتهایی | **مسیر API** یا **endpoint** | آدرسی که می‌شه بهش درخواست فرستاد — مثل `/api/users` |
| `payload` | محموله / بار | **داده درخواست** | اطلاعاتی که همراه درخواست می‌فرستیم |
| `token` | رمز / نشانه | **توکن** | کلید موقتی که بعد از ورود می‌گیری — مثل بلیط ورود |
| `session` | نشست / سشن | **نشست (session)** — یکدست استفاده کن | دوره‌ای که کاربر وارد شده و خارج نشده |
| `scope` | محدوده | **دامنهٔ دسترسی** | مشخص می‌کنه چه کارهایی با این توکن می‌شه کرد |
| `middleware` | میان‌افزار | **لایهٔ میانی** | کدی که بین درخواست کاربر و پردازش اصلی اجرا می‌شه |
| `webhook` | قلاب وب | **وب‌هوک** | وقتی سیستمی به یه URL خبر می‌ده که «یه اتفاق افتاد» |
| `cache` | کش / حافظهٔ پنهان | **cache** یا **حافظهٔ موقت** | ذخیرهٔ موقت نتیجه برای سریع‌تر کردن پاسخ |
| `reverse proxy` | پروکسی معکوس | **پروکسی** (یا توضیح: «واسطهٔ بین کاربر و سرور») | سرویسی که درخواست رو می‌گیره و به سرور اصلی می‌فرسته |
| `fallback` | بازگشتی / پشتیبان | **حالت جایگزین** | وقتی چیزی کار نکنه، این جایگزین اجرا می‌شه |

| `tenant` | مستأجر | **سازمان** یا **فضای مجزا** | هر مشتری/سازمان که پایگاه دادهٔ جداگانه داره |

## چطور از این فایل استفاده کنی

**وقتی خودت می‌نویسی** — قبل از publish یک بار جستجو کن: اصطلاح انگلیسیت در جدول هست؟

به‌جای «تصویر Docker ساخته شد» بنویس: «ایمیج Docker build شد».  
به‌جای «حملات تقاطعی» بنویس: «حملات بین‌سایتی (CSRF)».  
اگر در جدول نیست: انگلیسی + توضیح ساده در پرانتز.

**وقتی AI می‌نویسد** — در prompt بگو: «اصطلاحات طبق tech-terms-glossary». AI غالباً «تصویر» برای image می‌زند؛ این جدول اولویت دارد.

**یکدستی:** `session` و «نشست» یکی‌اند — در یک سند یکی را انتخاب کن و همان را نگه دار.
