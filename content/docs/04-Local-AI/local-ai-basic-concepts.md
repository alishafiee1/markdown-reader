## 📚 اول: آموزش مفاهیم پایه به زبان خیلی ساده

بیایید با چند اصطلاح کلیدی شروع کنیم:

---

### 🧠 LM (Language Model) = مدل زبانی
**مغز هوش مصنوعی** که متن می‌فهمد و تولید می‌کند.

> مثال ساده: مثل یک مغز که کد می‌نویسد، اما خودش به تنهایی نمی‌تواند فایل را باز کند یا دستوری اجرا کند. فقط حرف می‌زند و فکر می‌کند.

در سناریوی شما: **Qwen 2.5 Coder 32B** (روی کارت گرافیک) یا **Claude Sonnet** (از طریق اینترنت)

---

### 🔌 API (Application Programming Interface) = رابط برنامه‌نویسی
**پلی ارتباطی** که به نرم‌افزارها اجازه می‌دهد با مدل زبانی حرف بزنند.

> مثال ساده: مثل یک شماره تلفن که شما با آن با مغز (مدل) تماس می‌گیرید. شما سوال می‌پرسید، جواب می‌گیرید.

در سناریوی شما:
- **API ابری** → ارتباط با Claude از طریق اینترنت (پولی)
- **API محلی** → ارتباط با Qwen روی GPU خودتان (رایگان، آدرس `http://localhost:11434`)

---

### 🔧 MCP (Model Context Protocol) = پروتکل بافت مدل
**جعبه ابزار هوشمند** که به مدل یاد می‌دهد چطور از ابزارها استفاده کند.

> مثال ساده: مدل فقط می‌تواند حرف بزند. MCP یک جعبه ابزار به او می‌دهد که شامل: 🔍 جستجوگر، 📁 بازکننده فایل، 🖊️ ویرایشگر، 🐚 ترمینال. مدل یاد می‌گیرد کدام ابزار را کی استفاده کند.

در سناریوی شما: **TokenOS** یا **RagCode MCP** به مدل شما یاد می‌دهد چطور در کدبیس شما جستجو کند.

---

### 🏗️ Framework = چهارچوب کاری
**یک بسته آماده** شامل ابزارها، قوانین و ساختارهای از پیش ساخته شده.

> مثال ساده: مثل یک کیت مبلمان آماده که همه پیچ‌ها و مهره‌ها و نقشه سرهم کردن داخلش هست. شما فقط باید طبق دستورالعمل سرهم کنید.

در سناریوی شما: **Claude Code** خودش یک فریمورک کامل است که شامل مدل + ابزارها + قوانین اجرایی می‌شود.

---

## 🎯 دوم: نقش هر ابزار در سناریوی شما (بالاترین دقت و درک کد)

حالا که مفاهیم را فهمیدید، معماری نهایی شما برای **حداکثر دقت و درک کد** به این شکل است:

```
┌─────────────────────────────────────────────────────────────┐
│                    معماری نهایی شما                          │
└─────────────────────────────────────────────────────────────┘

لایه 1: لایه اجرای مدل (اجرای مغزها)
├── LM Studio یا Ollama
│   └── مدل محلی: Qwen 2.5 Coder 32B (روی RTX 3090 شما)
│
لایه 2: لایه درک پروژه (MCP Serverها)
├── TokenOS یا RagCode MCP
│   └── کار: ایندکس کردن کل کدبیس (درک ساختار توابع، کلاس‌ها، ارتباطات)
│
لایه 3: لایه ارکستراسیون (هماهنگ‌کننده اصلی)
├── Claude Code (از طریق API ابری - پولی)
│   └── کار: برنامه‌ریزی، تصمیم‌گیری، هماهنگی بین ابزارها
│
لایه 4: لایه بهینه‌سازی هزینه
├── Houtini LM
│   └── کار: کارهای ساده را به مدل محلی Qwen واگذار می‌کند
│
لایه 5: لایه رابط کاربری
├── Cursor (VS Code + هوش مصنوعی)
│   └── کار: جایی که شما کد می‌نویسید و با سیستم تعامل دارید
│
لایه 6: لایه حافظه بلندمدت
├── حافظه اختصاصی (Memory Bank)
│   └── کار: به یاد آوردن تصمیمات قبلی و معماری پروژه
```

### 📋 نقش هر ابزار در این معماری (به زبان خیلی ساده)

| ابزار | نقش | مثل چه چیزی در دنیای واقعی؟ |
| :--- | :--- | :--- |
| **Qwen 2.5 Coder** | مغز محلی رایگان که کد می‌نویسد | یک برنامه‌نویس جونیور که کارهای تکراری را سریع انجام می‌دهد |
| **Claude Sonnet (از API)** | مغز حرفه‌ای پولی که برنامه می‌ریزد | یک معمار ارشد که استراتژی و طراحی می‌کند |
| **Claude Code** | فریمورک کامل که خودش می‌داند چطور از ابزارها استفاده کند | یک ربات برنامه‌نویس که می‌تواند فایل باز کند، دستور اجرا کند، تصمیم بگیرد |
| **MCP Server (مثل TokenOS)** | جعبه ابزاری که به مدل یاد می‌دهد چطور کدبیس را بفهمد | یک کتابخانه عظیم با نمایه‌ساز که هر چیزی را می‌تواند پیدا کند |
| **Houtini LM** | یک مدیر هوشمند که تشخیص می‌دهد چه کاری ساده است (برود به Qwen) و چه کاری پیچیده است (برود به Claude) | یک منشی که نامه‌های ساده را خودش جواب می‌دهد و نامه‌های مهم را به مدیر ارشد می‌دهد |
| **Cursor** | محیطی که شما در آن کد می‌نویسید و چت می‌کنید | میز کار شما با قابلیت چت با دستیار هوشمند |
| **Memory Bank** | یک دفترچه یادداشت که تمام تصمیمات و معماری پروژه را ذخیره می‌کند | یک مستندساز خودکار که هر تصمیمی را ثبت می‌کند |

## 🏗️ سوم: معماری نهایی برای حداکثر دقت

### جریان کار (Flow)

```
شما در Cursor یک سوال می‌پرسید:
"لطفاً یک سیستم احراز هویت JWT به پروژه اضافه کن"

Cursor سوال را می‌فرستد به ← Claude Code (از طریق API)

Claude Code (مغز ارشد) تحلیل می‌کند:
"این کار به چند مرحله نیاز دارد. باید ابتدا بفهمم پروژه الان چه ساختاری دارد."

Claude Code از طریق MCP سوال می‌کند از ← TokenOS:
"توابع مرتبط با احراز هویت در کدبیس کجا هستند؟"

TokenOS (که قبلاً کل پروژه را ایندکس کرده) جواب می‌دهد:
"هیچ تابع احراز هویتی وجود ندارد. اما فایل‌های مسیر و کنترلر به این صورت هستند..."

Claude Code تصمیم می‌گیرد:
"مرحله 1: ایجاد مدل User. این کار ساده است، برو به Qwen محلی."

Houtini LM درخواست را می‌فرستد به ← Qwen 2.5 Coder (روی GPU شما)

Qwen کد مدل User را می‌نویسد و برمی‌گرداند.

Claude Code بررسی می‌کند و تصمیم می‌گیرد:
"مرحله 2: ایجاد میدلور JWT. این کار پیچیده است، خودم انجام می‌دهم."

Claude Code کد میدلور را می‌نویسد (از طریق API ابری خودش)

در نهایت، Claude Code تمام کدها را در فایل‌های مناسب در پروژه شما قرار می‌دهد.

همه این مراحل را در Memory Bank ثبت می‌کند تا جلسه بعد به یاد بیاورد.
```

## 🛠️ چهارم: تنظیمات نهایی برای پیاده‌سازی

### مرحله ۱: راه‌اندازی موتور محلی (برای Qwen)
```bash
# نصب LM Studio یا Ollama
# دانلود مدل Qwen 2.5 Coder 32B
ollama pull qwen2.5-coder:32b

# اجرا روی GPU
ollama run qwen2.5-coder:32b
```

### مرحله ۲: راه‌اندازی MCP Server (برای درک پروژه)
```bash
# نصب TokenOS در ریشه پروژه
npx -y tokenos .

# یا نصب RagCode MCP
npx -y @modelcontextprotocol/server-rag
```

### مرحله ۳: نصب و تنظیم Claude Code
```bash
# نصب Claude Code
npm install -g @anthropic-ai/claude-code

# تنظیم API Key (پولی)
export ANTHROPIC_API_KEY="sk-ant-..."

# اضافه کردن Houtini LM برای توزیع بار بین Claude و Qwen
claude mcp add houtini-lm -- npx -y @houtini/lm
```

### مرحله ۴: تنظیم Cursor
در فایل `~/.cursor/mcp.json`:
```json
{
  "mcpServers": {
    "tokenos": {
      "command": "npx",
      "args": ["-y", "tokenos", "."],
      "cwd": "/path/to/your/project"
    },
    "local-llm": {
      "url": "http://localhost:11434/v1",
      "model": "qwen2.5-coder:32b"
    }
  }
}
```

## 💡 خلاصه نهایی برای شما

با RTX 3090 24GB شما، **بهترین سناریو** این است:

1. **Claude Code** (پولی، از API) مسئول **برنامه‌ریزی و تصمیم‌گیری** می‌شود
2. **Houtini LM** به عنوان **مدیر توزیع بار** کارهای ساده را به **Qwen محلی** (رایگان) می‌دهد
3. **TokenOS MCP** به عنوان **جعبه ابزار جستجو** به مدل‌ها یاد می‌دهد چطور در پروژه شما جستجو کنند
4. **Memory Bank** همه تصمیمات را ثبت می‌کند تا در جلسات بعد به یاد بیاورد
5. **Cursor** فقط به عنوان **نمایشگر و رابط کاربری** عمل می‌کند

**نتیجه نهایی:** یک سیستم کدنویسی هوشمند با دقت بسیار بالا، درک کامل از پروژه، هزینه کنترل‌شده (با واگذاری کارهای ساده به مدل رایگان محلی)، و قابلیت دیباگ و اجرای خودکار دستورات.

آیا آماده‌اید مرحله اول (نصب LM Studio و دانلود Qwen) را شروع کنیم؟
